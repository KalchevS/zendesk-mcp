CHANGELOG — mcp-server-zendesk
================================

## v1.2.0 (2025-05-12)

### Added — Dual authentication support

The server now supports two authentication modes, auto-detected from env vars:

Mode 1 — API Token (simple, existing behavior):
  - Set ZD_EMAIL + ZD_API_TOKEN
  - Uses HTTP Basic Auth ({email}/token : {api_token})
  - Tokens don't expire

Mode 2 — OAuth Bearer Token (recommended for production):
  - Set ZD_OAUTH_ACCESS_TOKEN
  - Uses Authorization: Bearer header
  - Automatic token refresh on 401 if ZD_OAUTH_REFRESH_TOKEN, ZD_OAUTH_CLIENT_ID,
    and ZD_OAUTH_CLIENT_SECRET are also configured
  - Graceful fallback: if refresh fails, raises auth error to the client

### Changed

- config.py: Added OAuth settings (zd_oauth_access_token, zd_oauth_refresh_token,
  zd_oauth_client_id, zd_oauth_client_secret), auth_mode property, can_refresh_oauth property
- zendesk_client.py: ZendeskClient now auto-detects auth mode on open(),
  handles 401 with OAuth refresh in get/post/put, stores refreshed tokens in memory
- tools.py: bulk_solve_tickets_by_type now returns a clear error if ZD_EMAIL is not set
  (required for assignee filtering)
- .env.example: Documents both auth modes with commented examples

## v1.0.0 (2025-05-12)

Initial release as a standalone generic Zendesk MCP server.
Forked from zendesk-mcp and stripped of instance-specific tools.

### Architecture

- Async-first: All tool functions are async, using httpx.AsyncClient
- Dispatch map: TOOL_HANDLERS dict replaces 40-line if/elif chain (O(1) lookup)
- Lifespan management: HTTP client opened on startup, closed on shutdown
- Proper packaging: pyproject.toml with entry point `mcp-server-zendesk`
- Dual transport: stdio (subprocess) and HTTP/SSE (Docker/network)

### Improvements over original codebase

1. Dispatch map replaces if/elif chain
   - TOOL_HANDLERS dict for O(1) tool routing
   - Eliminates 40+ lines of repetitive dispatch code

2. Error masking for production
   - MASK_ERRORS=true in .env hides internal error details from AI clients
   - Prevents leaking URLs, emails, or stack traces in production

3. Lifespan/shutdown hook
   - stdio mode: run_stdio() opens/closes the async HTTP client
   - HTTP mode: Starlette lifespan context manager handles startup/shutdown
   - No more dangling connections on server exit

4. Async tools + AsyncClient
   - All tool functions are async def
   - httpx.AsyncClient replaces sync Client
   - Multiple SSE clients can connect concurrently without blocking

5. Input validation centralized
   - _validate_ticket_query() for query guards
   - _validation_error() for consistent error payloads
   - _zendesk_error_response() respects MASK_ERRORS setting

6. Rate limit retry fixed
   - Custom _is_retryable() predicate: retries on 429 and 5xx
   - Both GET and PUT now properly retry on rate limits
   - Previous version raised ZendeskError before tenacity could catch it

7. pyproject.toml added
   - Proper Python packaging with hatchling build backend
   - Entry point: `mcp-server-zendesk` command
   - Pinned dependencies with minimum versions

### Removed (instance-specific tools)

- search_new_starters (HR onboarding)
- search_departures (HR offboarding)
- get_departure_summary (HR statistics)
- solve_visa_news_tickets (hardcoded custom field IDs)
- search_escalated_tickets (hardcoded tag name)
